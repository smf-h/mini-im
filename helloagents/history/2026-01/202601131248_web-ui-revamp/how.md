# Web 端 UI 重构方案（How）

## 1) 设计方向（推荐）
**风格关键词：现代极简 / 高对比可读 / 轻玻璃点缀 / WeChat 绿作为强调色。**

- 亮色：大面积干净浅底 + 轻边框 + 轻阴影，强调“信息密度与可读性”。
- 暗色（可选）：深海蓝灰底 + 更克制的阴影 + 以绿色作为少量高亮。

> 说明：本方案优先“不引入重量级 UI 框架”，通过 Tokens + 组件化实现一致性，避免依赖膨胀与风格割裂。

## 2) 信息架构与布局草图

### 桌面端（≥1024px）：三栏主结构 + 可选信息侧栏
```
┌──────────────┬─────────────────────┬──────────────────────────────────────┐
│ 左导航 Rail    │ 列表 Panel           │ 主内容 Main                           │
│ - 头像/状态    │ - 搜索/筛选          │ - 顶部：标题/状态/动作                 │
│ - 会话/联系人  │ - 列表项(会话/联系人) │ - 中部：内容(聊天/资料/动态等)           │
│ - 群聊/动态    │ - 未读/置顶/草稿      │ - 底部：输入区/操作区（聊天时）          │
│ - 设置         │                     │ + 可选：右侧 Drawer 展示群成员/设置等     │
└──────────────┴─────────────────────┴──────────────────────────────────────┘
```

### 平板/小屏（768–1023px）：两栏（列表 + 内容），信息侧栏改为 Drawer
```
┌─────────────────────┬──────────────────────────────────────┐
│ 列表 Panel           │ 主内容 Main                           │
└─────────────────────┴──────────────────────────────────────┘
```

### 移动端（<768px）：底部 Tab + 进入聊天全屏
```
┌──────────────────────────────────────┐
│ 顶栏：标题/返回/动作                   │
├──────────────────────────────────────┤
│ 内容区：列表/动态/聊天流               │
├──────────────────────────────────────┤
│（聊天时）输入区：多行 + 发送 + 附件入口 │
└──────────────────────────────────────┘
底部 Tab：会话 / 联系人 / 群聊 / 动态 / 设置
```

## 3) Design Tokens（落地到 CSS 变量）
建议将现有 `app.css` 的变量扩展为更体系化的 tokens（命名更语义化，支持主题切换）：
- 颜色：`--color-bg`, `--color-surface`, `--color-text`, `--color-muted`, `--color-border`, `--color-primary`, `--color-danger`
- 形状：`--radius-sm/md/lg`
- 阴影：`--shadow-sm/md/lg`
- 字体：`--font-size-12/14/16/18`, `--font-weight-regular/medium/semibold`
- 间距：`--space-4/8/12/16/20/24/32`

## 4) 关键组件与交互（统一体验）
- 导航：左侧 Rail（图标+Badge），移动端 TabBar。
- 列表：统一 `ListItem`（左 Avatar / 中标题+摘要 / 右时间+未读），支持 hover/focus 但避免 scale 导致抖动。
- 聊天：消息气泡（我方/对方/系统），时间分隔；“重要消息”高亮；输入区支持多行与发送快捷键。
- 资料与设置：使用同一套 `Card/Section` 结构，避免各页风格分裂。
- Toast：保持当前“轻玻璃浮层”方向，但与 Tokens 对齐，统一颜色与阴影。

## 5) 风格备选（如你想更大胆）
- **A / 推荐：Modern Minimal（干净耐看，像生产级）**
- **B：Soft Glass（更通透，更偏“设计感”）**
- **C：Dark Neon（深色 + 霓虹强调，氛围更强）**

## 6) 影响范围（实现时预计改动）
- `frontend/src/views/*`：主要页面布局与样式重构（会话、聊天、群聊、联系人、动态、登录、设置、资料）。
- `frontend/src/components/*`：新增/强化布局组件（Rail/Panel/Header/Drawer/TabBar 等）。
- `frontend/src/styles/app.css`：Tokens + 主题能力（亮/暗）。
- `frontend/src/router.ts`：配合桌面三栏布局，可能需要调整部分路由嵌套与默认重定向逻辑。
