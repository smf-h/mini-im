# Web 端 UI 重构方案（Why）

## 背景
`mini-im` 的 Web 端（`frontend/`）已经具备核心能力：登录、会话列表、单聊/群聊、联系人、群组、朋友圈、资料页与设置等。但目前的视觉语言与信息架构更偏“功能优先”，在桌面端信息密度、交互效率、视觉一致性与移动端适配方面仍有明显提升空间。

## 目标
1. **更像“生产级 IM”**：清晰的信息层级、稳定的交互反馈、可预期的布局与快捷操作。
2. **桌面端更高效**：三栏布局（导航/列表/内容）减少来回跳转；可在列表与聊天之间快速切换。
3. **移动端更顺手**：底部 Tab + 全屏聊天；触控目标更大；输入区与滚动行为更稳定。
4. **视觉一致与可维护**：统一 Design Tokens（颜色/字号/间距/圆角/阴影），沉淀为可复用组件。
5. **可访问性与可读性**：对比度、焦点态、键盘可用性、可理解的空状态与错误状态。

## 成功标准（验收口径）
- 桌面端：会话/联系人/群聊等“列表型页面”在左侧固定；聊天在右侧主区域，支持在不离开列表的情况下切换会话。
- 移动端：底部 Tab 导航；进入聊天后全屏沉浸；返回逻辑明确，输入框不遮挡内容。
- 组件一致性：按钮、输入框、列表项、Badge、Avatar、Modal/Drawer/Toast 的样式与交互统一。
- 主题能力：默认亮色；预留暗色主题（可开关），并能自动适配 `prefers-color-scheme`。
